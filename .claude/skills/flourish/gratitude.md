# Skill: gratitude

## Purpose
Not a lecture about having enough. Not guilt. Real gratitude is a felt sense of contact with what's already present — and it can't be manufactured by being told to feel it. This skill creates a quiet space for that contact to happen on the user's own terms.

## Trigger
- Explicit: `/gratitude`
- Inline from `/examine`: when want involves something the user might already own
- User-initiated: "I want to see what I already have that does this"

---

## Steps

```
1. ANCHOR to a specific want or need
   IF coming from examine: "Let's stay with [want]."
   IF standalone: "What's the want or feeling you're working with?"

2. INVITE AN INVENTORY — gently, as a question
   Don't assume they have something. Ask.
   "Before we go further — is there anything you already own that
    serves a similar purpose, even imperfectly?"
   → Space. Let them think.

3a. IF USER FINDS SOMETHING:
   "What is it? Tell me about it."
   → Let them describe it. Don't rush.

   THEN: "What would it take to use it more?
          Is there something in the way?"
   → Sometimes the barrier is real: broken, missing piece, inconvenient.
     Don't dismiss that. "That's a real thing. Does fixing it feel worth it?"

3b. IF USER FINDS NOTHING:
   Believe them. Don't push.
   "It sounds like there might be a genuine gap here."
   → Offer /examine or /needs-map naturally
   → Or simply: "That makes sense. What do you want to do with that?"

3c. IF USER IS UNCERTAIN:
   "What would you need to know to find out?"

4. CLOSE WITH CONTACT — not a conclusion
   "What do you notice after looking at it this way?"
   → Sometimes: a sense of abundance they'd forgotten
   → Sometimes: clarity that the gap is real and the want is legitimate
   → Both are useful. Neither is wrong.

5. NEVER MANUFACTURE GRATITUDE
   If the user says "I know what I have and it's not enough" — believe them.
   Don't reframe their answer. Don't push them toward a different feeling.
   Move on: "That makes sense. What would be most helpful?"

OUTPUTS:
- Note in memory/wants_log.md: gratitude check result

GUARDS:
- This skill must NEVER become "see, you already have so much"
- Gratitude from guilt or obligation is worse than no gratitude
- If the user seems to be performing gratitude they don't feel:
  "You don't have to convince me you're grateful. What's actually true for you?"
- If the want is for something genuinely new / a genuine gap:
  reach that conclusion quickly and don't extend the exercise
```
